## 1. 项目适合做什么

这个项目适合做电商平台常见接口测试，例如：

- 登录接口
- 商品接口
- 订单接口
- 营销接口
- 库存接口
- 资金对账
- 财务结算

## 2. 第一次使用怎么做

### 2.1 下载代码

把项目拉到本地后，进入项目根目录。

### 2.2 安装 Python 依赖

```bash
pip install -r requirements.txt
```

### 2.3 安装 Allure

如果本机还没有 Allure，可以按团队常用方式安装。

安装完成后，执行下面命令检查：

```bash
allure --version
```

### 2.4 执行测试

执行全部用例：

```bash
pytest
```

只执行登录模块：

```bash
pytest -m login
```

执行指定环境：

```bash
pytest --env=uat
```

## 3. 多账号怎么用

## 4. 不同业务模块怎么绑定不同账号

框架已经内置了这些登录夹具：

- `order_login_token`
- `product_login_token`
- `refund_login_token`

用法示例：

```python
def test_order_list(http_client, order_login_token):
    pass

def test_product_list(http_client, product_login_token):
    pass
```

如果账号还是占位数据，框架会自动跳过，不会误跑。

## 6. 新增接口怎么做

下面用最常见的流程说明。

### 第一步：确认接口信息

至少要准备好这些内容：

- 接口地址
- 请求方式
- 请求头
- 入参
- 返回值
- 断言点

### 第二步：在 `data` 目录加测试数据

例如新增商品查询接口，可以新建：

- `data/product/product_list.yaml`

数据建议按“一个接口一个 yaml 文件”管理。

### 第三步：在 `api` 目录管理接口路径

例如新增：

- `api/product_api.py`

把路径单独写出来，后面统一维护更方便。

### 第四步：在 `testcases` 目录写用例

例如新增：

- `testcases/test_product.py`

用例只做三件事：

- 读取数据
- 发请求
- 做断言

不要把太多复杂逻辑写到用例里。

## 8. 订单列表示例怎么用

当前项目已经接入一个真实订单列表示例：

- 接口：`/saas/orderCenter/getCashierOrderLsit`
- 接口：`/saas/getShopDetail`
- 接口：`/shop/order/addShopOrderBySeller`
- 接口：`/saas/order/queryNonPayOrder`
- 接口：`/saas/orderCenter/queryDetail`
- 接口：`/saas/payment/cashPayment`
- 请求方式：`POST`
- 额外请求头：`platform: saas-jr`
- 鉴权方式：自动带 `jwtToken`

示例位置：

- [order_case.yaml](/Users/xueguanglai/Desktop/work/CSJKtest/data/order/order_case.yaml)
- [test_order.py](/Users/xueguanglai/Desktop/work/CSJKtest/testcases/test_order.py)

执行这个用例时，流程是：

1. 先调用登录接口
2. 提取登录返回里的 `jwtToken`
3. 把 `jwtToken` 自动放到请求头
4. 调用生成订单接口并提取 `orderId`
5. 后续接口通过 `{{orderId}}` 自动关联使用

## 10. 出现问题怎么排查

### 7.1 依赖安装失败

先检查：

- Python 版本是否正常
- `pip` 是否可用
- 网络是否正常

### 7.2 接口请求失败

先检查：

- 环境地址是否正确
- 请求头是否正确
- 参数是否正确
- 账号是否有效
- 测试环境服务是否可用

### 7.3 报告打不开

先检查：

- Allure 是否安装
- `reports/allure-results` 是否有结果文件

## 11. 团队维护建议

- 接口地址统一放 `api`
- 测试数据统一放 `data`
- 公共方法统一放 `common`
- 每个模块独立写测试文件
- 不要把写死数据散落在代码里
- 一条用例只验证一个核心目标

## 12. 失败通知怎么配

当前项目已经支持测试失败后发钉钉通知。

配置位置：

- [project.yaml](/Users/xueguanglai/Desktop/work/CSJKtest/config/project.yaml)

可配置内容：

- `dingtalk_webhook`
- `dingtalk_keyword`
- `dingtalk_image_url`

说明：

- 只有失败或异常时才发送
- 图片推送走的是钉钉 `markdown` 图片方式
- 图片地址必须是公网可访问地址，本地图片不能直接在钉钉里展示
