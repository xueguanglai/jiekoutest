# 新增业务模块标准流程

这份文档是给团队共建时直接照着做的，不讲复杂理论，只讲实际操作。

## 1. 新增一个业务模块时要做什么

标准动作只有 5 步：

1. 准备账号
2. 准备接口数据
3. 准备接口路径
4. 新增测试用例
5. 执行并看报告

## 2. 第一步：准备账号

先判断这个业务模块要不要单独账号。

如果要，就到：

- [account.yaml](/Users/xueguanglai/Desktop/work/CSJKtest/data/account/account.yaml)

新增一个角色。

例如新增库存模块账号：

```yaml
stock_user:
  title: 库存模块账号
  areaCode: "86"
  phone: "请替换成库存账号"
  password: "请替换成库存密码"
  sliderCode: ""
  signKey: "请替换成真实 signKey"
  loginType: 2
```

## 3. 第二步：在 conftest.py 加登录夹具

如果是新角色，建议补一个专用夹具。

例如：

```python
@pytest.fixture(scope="session")
def stock_login_token(role_login):
    return role_login("stock_user")
```

这样后面库存模块用例就可以直接复用。

## 4. 第三步：准备接口数据

在 `data` 目录新增模块目录。

例如：

- `data/stock/stock_case.yaml`

建议一个模块一个 yaml 文件起步，后面再按接口拆分。

示例：

```yaml
- title: 查询库存列表
  method: post
  url: /api/stock/list
  body:
    pageNum: 1
    pageSize: 10
  expected_status_code: 200
  expected_business_codes:
    - 0
    - 200
  expected_keys:
    - code
```

## 5. 第四步：准备接口路径

在 `api` 目录新增模块接口文件。

例如：

- `api/stock_api.py`

先只写最核心的接口路径，不要一开始封装太重。

## 6. 第五步：新增测试文件

在 `testcases` 目录新增测试文件。

例如：

- `testcases/test_stock.py`

示例写法：

```python
import allure
import pytest

from common.path_util import DATA_DIR
from common.response_util import validate_and_extract
from common.yaml_util import read_yaml


case_data = read_yaml(DATA_DIR / "stock" / "stock_case.yaml")


@allure.feature("库存模块")
class TestStock:
    @allure.story("库存接口")
    @pytest.mark.parametrize("case", case_data)
    def test_stock_business(self, http_client, stock_login_token, case):
        response = http_client.send(
            method=case["method"],
            url=case["url"],
            headers=case.get("headers", {}),
            json=case["body"],
        )
        validate_and_extract(response, case)
```

## 7. 第六步：执行验证

执行当前模块：

```bash
pytest testcases/test_stock.py
```

如果要看报告，就打开 `reports` 目录下最新生成的 HTML 报告。

## 8. 团队统一要求

- 一个模块尽量绑定一个角色账号
- 一个接口尽量一条数据一条测试意图
- 先保证跑通，再补细化断言
- 复杂逻辑放 `common`
- 业务差异放 `data`

## 9. 新增模块时最容易漏掉什么

- 忘了加账号
- 忘了换掉占位账号
- 忘了补请求头
- 忘了补业务码断言
- 忘了提取上下游关联字段
