# 常见问题排查手册

这份文档是给团队排错用的，重点是遇到问题先查什么。

## 1. 用户认证失败

先检查：

- 登录接口是否成功
- `jwtToken` 是否提取成功
- 请求头里是否带了 `Authorization`
- 账号是否过期或无权限

重点文件：

- [login_service.py](/Users/xueguanglai/Desktop/work/CSJKtest/common/login_service.py)
- [project.yaml](/Users/xueguanglai/Desktop/work/CSJKtest/config/project.yaml)

## 2. 后续接口拿不到 orderId

先检查：

- 生成订单接口是否成功
- 真实返回里订单号字段路径是不是 `data.orderId`
- 下游接口是不是写了 `{{orderId}}`

## 3. 报 DNS 或网络错误

例如：

- 域名无法解析
- 连接超时
- 环境不可访问

先检查：

- 本机网络
- VPN 或办公网络
- 测试环境服务状态
- 域名是否只能内网访问

## 4. 用例被跳过

最常见原因：

- 账号还是占位数据
- 模板 case 还没有替换真实接口

先检查：

- [account.yaml](/Users/xueguanglai/Desktop/work/CSJKtest/data/account/account.yaml)
- 对应 yaml 是否还写了 `skip: true`

## 5. 生成的 HTML 报告没有内容

先检查：

- 测试是否真的执行了
- `reports` 目录下是否有最新报告
- 当前解释器是否正常执行 `pytest`

## 6. 钉钉通知没发出去

先检查：

- webhook 是否正确
- 自定义关键词是否命中
- 当前运行是否真的有失败
- 网络是否允许访问钉钉接口

配置文件：

- [project.yaml](/Users/xueguanglai/Desktop/work/CSJKtest/config/project.yaml)

## 7. 新人最容易犯的错误

- 把账号写死到代码里
- 接口地址写死在测试文件里
- 返回值字段路径写错
- 直接复制别的模块，不改夹具
- 把占位账号忘了换掉
